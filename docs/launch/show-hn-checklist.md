# Show HN Launch Checklist

## Before posting

- make sure `main` is clean
- make sure README top section matches the current launch message
- make sure the repo link resolves correctly
- make sure the first demo GIF is either ready or intentionally omitted
- keep the first comment ready to paste

## At posting time

- post the title exactly as written in `show-hn-post.md`
- paste the body without extra formatting noise
- add the first comment immediately after posting
- stay available to reply while the post is fresh

## After posting

- answer questions quickly and concretely
- prefer short technical replies over marketing language
- note repeated objections and feed them back into README or launch docs
- capture good spec examples or failure reports for follow-up issues
